# 丹书 Style Reference

## Page Archetypes

### Page 1 · Thesis + Metrics

Use when the document needs an immediate point of view.

- Header: vermilion left rule, large title, small author/context block.
- Body: one distilled thesis paragraph.
- Visualization: a low-amplitude timeline, stepped sequence, or compact evidence chart.
- Metrics: 4-5 compact number-first tiles.
- Close: one ivory callout with a memorable sentence.

### Page 2 · Structure Map

Use when the source contains choices, topics, roles, or scenarios.

- Three cards usually work better than a dense table.
- Use local arcs, bracket curves, or enclosure paths to relate cards. Do not place the same pale wave behind every card by habit.
- Each card should contain: label, title, question, 2-3 observable actions.
- Keep cards visually equal, but let the copy carry hierarchy.

### Page 3 · Judgment / Scoring / Action

Use when the reader needs to know what “good” means.

- Put the most important hard requirement in a prominent tile.
- Use an S-curve only for actual sequence or transformation. Use bars, thresholds, or a decision path for scoring and judgment.
- Keep each anchor dot and its text within 34-52pt.
- Use a stacked bar for scoring weights.
- End with a short warning or principle.

## Curves

Curves are structural lines, not ornaments.

Before drawing, name the curve's job in one word: `time`, `route`, `flow`, `comparison`, `grouping`, `attention`, or `boundary`.

- Stroke: `#DDBBB2`, 1.1-1.6pt in PDF.
- A focal curve may use `#A33A2A`, but only one per page.
- Keep control points smooth; avoid zigzags unless the source is explicitly about interruption.
- It is acceptable for pale curves to pass behind text, but not through tiny labels or numbers.
- If a numbered dot is separated from its text, move the text, not the dot.
- If removing the curve would not change the reader's understanding, delete it or replace it with whitespace, a divider, bracket, bar, or grid.

### Curve Grammar

Use different silhouettes for different jobs. A beautiful curve is not enough; its arc must encode the relation.

| Job | Use | Shape | Avoid |
| --- | --- | --- | --- |
| `time` | dates, stages, before/after | low horizontal wave, stepped line, or sparse timeline | dramatic S-curve |
| `route` | fieldwork path, street walk, user journey | map-like path with checkpoints and uneven spacing | perfectly even decorative wave |
| `flow` | process, causality, transformation | S-curve or directional path with numbered anchors | dots floating far from labels |
| `comparison` | two logics, before vs after, amplitude vs frequency | split baseline, opposing arcs, or two-column divider | one shared wave implying false continuity |
| `grouping` | cards, categories, options | local arcs, brackets, enclosure curve, or subtle card alignment | one long curve repeated behind all cards |
| `attention` | one striking conclusion or threshold | short underline arc, circled threshold, or focal hook | full-width background flourish |
| `boundary` | requirement, cutoff, threshold | gate line, bracket, vertical rule, or threshold bar | soft wave that hides the cutoff |

### Anti-Repetition Rule

Across a 3-page brief, adjacent pages must not use the same curve family.

Good sequence:

- Page 1 evidence: low timeline with small checkpoint dots.
- Page 2 structure: local arcs or brackets around three cards.
- Page 3 scoring: stacked bar plus threshold gate; no background wave.

Weak sequence:

- Page 1 wave timeline.
- Page 2 same wave behind cards.
- Page 3 same wave under scoring dots.

When a document has multiple diagrams, create a tiny "curve inventory" before layout:

```text
P1: time -> low timeline
P2: topics -> local card arcs
P3: scoring -> threshold bar + small attention hook
```

If two pages share the same silhouette, revise one before rendering.

## Chart Selection

- Time window: low timeline, stepped timeline, or restrained line chart.
- Milestone sequence: S-curve only if the sequence has a real transformation; otherwise use a checkpoint path.
- Percent scoring: stacked horizontal bar.
- Topic options: card map.
- Before/after or year-to-year data: line chart.
- Category comparison: bar chart.
- Proportions: donut only if it clarifies a true whole.

## Density

Aim for 60-80% useful fill on body pages.

- Too sparse: add a diagram or merge sections.
- Too crowded: shorten copy before shrinking type.
- Last page may be quieter if it carries the conclusion.

## Metadata Block

For class handouts and teaching reports:

- Keep the teacher name smaller than the title.
- Include class names in metadata when provided.
- Example:
  - label: `教师`
  - name: `陈雁` at 11-13pt
  - context: `财政2505 · 金融2403`
  - unit: `经济学院金融教研室`
